Find the existing `.matrix-theme {` block in globals.css. Inside it, after the existing `background-color` and `color` declarations, add the four new tokens:

```css
/* Phase 32: Matrix Color Harmony tokens */
--matrix-green-subtle: #00FF4108;   /* ~3% opacity — very faint glow bg */
--matrix-green-border: #00FF4133;   /* ~20% opacity — border tint */
--matrix-scan-line: #00FF410D;      /* ~5% opacity — CRT scan line (Phase 33) */

```

CRITICAL: These MUST go inside `.matrix-theme {}` — NOT inside `:root`. The three existing `:root` tokens (`--matrix-green`, `--matrix-green-dim`, `--matrix-green-ghost`) stay untouched. Dashboard routes never have `.matrix-theme`, so these tokens are invisible to them.

Never use `@theme inline` to register these — that would make them global Tailwind utilities polluting the dashboard.

**section-label.tsx — Create new component:**

Create `apps/web/components/portfolio/section-label.tsx`:

```tsx
interface SectionLabelProps {
  label: string;
}

export function SectionLabel({ label }: SectionLabelProps) {
  return (
    <div
      className="font-mono text-xs text-[var(--matrix-green)] tracking-widest uppercase mb-2"
      aria-hidden="true"
    >
      {'>'} {label.toUpperCase().replace(/\s+/g, '_')}
    </div>
  );
}
```

`aria-hidden="true"` is correct: the label is decorative — the `<h2>` below it provides the accessible heading. Screen readers skip the `> PROFESSIONAL_SUMMARY` prefix.

No `'use client'` directive — this is a pure presentational Server Component.
